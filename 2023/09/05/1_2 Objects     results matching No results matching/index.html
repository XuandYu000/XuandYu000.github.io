Instance Variables and Object Instantiation

Not all dogs are alike. Some dogs like to yap incessantly, while others bellow sonorously(善良的), bringing joy to all who hear their glorious call. Often, we write programs to mimic(模仿) features of the universe we inhabit, and Java's syntax was crafted(制作) to easily allow such mimicry.

One approach to allowing us to represent the spectrum of Dogdom would be to create separate classes for each type of Dog.

As you should have seen in the past, classes can be instantiated(实例化), and instances can hold data. This leads to a more natural approach, where we create instances of the Dog class and make the behavior of the Dog methods contingent upon(取决于) the properties of the specific Dog. Some key observations and terminology:

  • An Object in Java is an instance of any class. Java中的一个对象object是其中一个类的实例
  • The Dog class has its own variables, also known as instance variables or non-static variables. These must be declared inside the class, unlike languages like Python or Matlab, where new variables can be added at runtime. Dog类有自己的变量也被称作实例变量或者非静态变量.这些必须在类中被声明不像python或Matlab中新的变量可以在运行时被加入
  • The method that we created in the Dog class did not have the static keyword. We call such methods instance methods or non-static methods. 创建Dog类并没有加入static关键字。我们称这样的方法为实例方法或非静态方法
  • To call the makeNoise method, we had to first instantiate a Dog using the new keyword, and then make a specific Dog bark. In other words, we called d.makeNoise() instead of Dog.makeNoise(). 为了调用makeNoise方法我们必须通过new关键字实例化他,之后会创建一个具体的Dogbark。换句话说,我们调用d.makeNoise()而不是Dog.makeNoise()
  • Once an object has been instantiated, it can be assigned to a declared variable of the appropriate type, e.g. d = new Dog(); 一旦对象被实例化,它可以被分配一个相同种类的变量名
  • Variables and methods of a class are also called members of a class. 一个类中的变量和方法也被称作类中的成员
  • Members of a class are accessed using dot notation. 类成员可被使用.调用

Static Variables

It is occasionally useful for classes to have static variables. These are properties inherent to the class itself, rather than the instance. For example, we might record that the scientific name (or binomen) for Dogs is "Canis familiaris": 类中存在静态变量偶尔有用.这些是类自身继承的属性,而不是实例.

Static variables should be accessed using the name of the class rather than a specific instance, e.g. you should use Dog.binomen, not d.binomen.

While Java technically allows you to access a static variable using an instance name, it is bad style, confusing, and in my opinion an error by the Java designers. 尽管java技术上允许你使用实例名称访问静态变量,但这是错误的,让人困惑,并且在我看来是java设计者的错误

public static void main(String[] args)

With what we've learned so far, it's time to demystify(神秘面纱) the declaration we've been using for the main method. Breaking it into pieces, we have:

  • public: So far, all of our methods start with this keyword.
  • static: It is a static method, not associated with any particular instance.
  • void: It has no return type.
  • main: This is the name of the method.
  • String[] args: This is a parameter that is passed to the main method.
