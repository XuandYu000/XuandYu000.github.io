4_4 Java libraries and packages

Abstract Data Types (ADTS)

Despite not talking about them explicitly, we have actually seen a few Abstract Data types already in class!

Two examples are List61B and Deque. Let's hone in on Deque.

We have this interface deque that both ArrayDeque and LinkedListDeque implement. What is the relationship between Deque and its implementing classes? Well, deque simply provides a list of methods (behaviors):

These methods are actually implemented by ArrayDeque and LinkedListDeque.

In Java, Deque is called an interface. Conceptually, we call deque an Abstract data type. Deque only comes with behaviors, not any concrete ways to exhibit those behaviors. In this way, it is abstract.

Java Libraries

Java has certain built-in Abstract data types that you can use. These are packaged in Java Libraries.

The three most important ADTs come in the java.util library:

  • List: an ordered collection of items
    • A popular implementation is the ArrayList
  • Set: an unordered collection of strictly unique items (no repeats)
    • A popular implementation is the HashSet
  • Map: a collection of key/value pairs. You access the value via the key.
    • A popular implementation is the HashMap

Finish the exercises below by using the above three ADT's. Reading the documentations linked above will help immensely.

These three ADT's all extend from the Collection Interface. The collection interface is super vague. Java says collections"represent a group of objects, known as its elements".

Java vs Python

But, Java has its upsides too! It gives you a lot of choices and freedom. For example, python only has one dictionary type which is declared using curly brackets {}. With Java, if you want to use an ADT like a Map, you can choose what kind of map you want: a Hashmap? a Treemap? etc. 但是Java也有自己的好处! 它给你选择的自由. 比如, python只有一种字典, 这由花括号{}生命. 在Java中, 如果你想使用ADT, 你可以从Hashmap和Treemap中选择你想要的类型.

We like Java in 61B! Here are some reasons why:

  • Arguably, takes less time to write programs(减少写程序的时间), due to features like:

    • Static types (provides type checking and helps guide programmer).静态类型(提供类型检查并且帮助程序员理解)
    • Bias towards interface inheritance leading to cleaner subtype polymorphism.(通过接口继承来实现简洁的子类型多态)
    • Access control modifiers make abstraction barriers more solid.(访问控制修饰符使抽象障碍更加坚固。)

  • More efficient code(更有效率的代码), due to features like:

    • Ability to have more control over engineering tradeoffs.
    • Single valued arrays lead to better performance.

  • Basic data structures more closely resemble underlying hardware:

    • Would be weird to do ArrayDeque in Python, since there is no need for array resizing. However, in hardware (see 61C), variable length arrays don’t exist.

Abstract classes

We've seen interfaces that can do a lot of cool things! They allow you to take advantage of interface inheritance and implementation inheritance. As a refresher, these are the qualities of interfaces: 我们已经看到接口可以做很多很酷的事! 你会在接口继承和实现继承中体会实际的便利. 作为一个初学者, 这里有些接口的性质你需要知道:

  • All methods must be public.(所有的方法必须是公共的)
  • All variables must be public static final.(所有的变量必须是常量)
  • Cannot be instantiated(不能被实例化)
  • All methods are by default abstract unless specified to be default(所有的方法都默认是抽象类型除非被default指明)
  • Can implement more than one interface per class(每个类可以实现多个接口)

We will now introduce a new class that lies somewhere in between interfaces and concrete classes: the abstract class. Below are the characteristics of abstract classes: 我们将会介绍一种介于接口和实际类的新的类: 抽象类. 下面是抽象类的特征

  • Methods can be public or private(方法可以是公共或私有的)
  • Can have any types of variables(可以有任意类型的变量)
  • Cannot be instantiated(不能被实例化)
  • Methods are by default concrete unless specified to be abstract(方法默认是明确的除非用abstract指明)
  • Can only implement one per class(每个类只能实现一个抽象类)

Basically, abstract classes can do everything interfaces can do and more. 可以说, 抽象类型可以做任何接口可以做的事甚至更多

When in doubt, try to use interfaces in order to reduce complexity. 当陷入疑惑时, 尝试使用接口, 这样做是为了减小复杂性

Packages

Package names give give a canonical name for everything. Canonical means a unique representation for a thing. 包名给所有事物命名. 典范意味着对一件事都有一个独特的名字.
